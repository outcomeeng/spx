# PLAN

## Reconcile the CLI command bindings to the CLI composition convention

[`spx/14-cli-composition.adr.md`](../../14-cli-composition.adr.md) requires every command domain's pure computation to live under `src/domains/{domain}/` with no Commander or process access, its process-agnostic I/O handlers under `src/commands/{domain}/`, and its Commander registration descriptor at `src/interfaces/cli/{domain}.ts`, with dependency flowing `interfaces/cli → commands → domains` and the composition root iterating a static descriptor registry.

The reconciliation rolls out incrementally — one descriptor relocation per PR, with the static registry introduced once every descriptor lives in the CLI-interface layer (the registry imports all of them). Each step runs through `/spec-tree:applying` with the TypeScript architecture and code audit gates.

### Done

- The `session` descriptor moved from `src/domains/session/index.ts` to `src/interfaces/cli/session.ts`; `src/cli.ts` imports it from the new location; `src/domains/session/` holds only pure modules. Behaviour and the node's `node bin/spx.js` evidence are unchanged.
- The `claude` descriptor moved from `src/domains/claude/index.ts` to `src/interfaces/cli/claude.ts` (PR #80). `src/domains/claude/` is now empty (directory removed). The relative `../types` import was rewritten to `@/domains/types`.
- The `config` descriptor moved from `src/domains/config/index.ts` to `src/interfaces/cli/config.ts` (PR #83). `src/domains/config/` retains only `root.ts` (pure `resolveProductDir` helper). Relative imports rewritten to `@/domains/types` and `@/domains/config/root`. The local ADR `spx/16-config.enabler/21-config-cli.enabler/21-cli-composition.adr.md` updated to reflect the new location.
- The `spec` descriptor moved from `src/domains/spec/index.ts` to `src/interfaces/cli/spec.ts`. `src/domains/spec/` retains only `fixture-writer-methods.ts` (pure spec-tree fixture metadata). Relative `../types` import rewritten to `@/domains/types`. Test import in `spx/31-spec-domain.enabler/76-spec-cli-contract-tests.enabler/tests/spec-cli-contract.scenario.l2.test.ts` updated to `@/interfaces/cli/spec`. No local ADR in the spec-domain subtree referenced the old path.
- The `validation` descriptor moved from `src/domains/validation/index.ts` to `src/interfaces/cli/validation.ts`. `src/domains/validation/` is now empty (directory removed). The relative `../types` import was rewritten to `@/domains/types`. The three co-located test imports in `spx/41-validation.enabler/` and the four testing-harness/generator imports under `testing/` were retargeted to `@/interfaces/cli/validation`. The broken markdown link in `spx/local/typescript.md` was updated to point at the new file.
- The `audit` descriptor moved from `src/domains/audit/cli.ts` to `src/interfaces/cli/audit.ts`. The reconciliation also created the missing `src/commands/audit/` tier: `runVerifyCommand` is now exported from `src/commands/audit/verify.ts`, the descriptor at `src/interfaces/cli/audit.ts` imports the handler through that path, and `src/domains/audit/` now contains only pure modules. Three test imports in the audit subtree were retargeted — `runVerifyCommand` consumers (`spx/36-audit.enabler/tests/audit.scenario.l1.test.ts`, `spx/36-audit.enabler/43-audit-config.enabler/tests/audit-config.compliance.l1.test.ts`, `spx/36-audit.enabler/76-audit-cli.enabler/tests/audit-cli.scenario.l1.test.ts`) read it from `@/commands/audit/verify`, and the descriptor consumer (`audit-cli.scenario.l1.test.ts`) reads `auditDomain` from `@/interfaces/cli/audit`. The local ADRs `spx/36-audit.enabler/21-audit-module-structure.adr.md` and `spx/36-audit.enabler/76-audit-cli.enabler/21-audit-cli.adr.md` were updated to reflect the three-layer split.
- The static descriptor registry now lives at `src/interfaces/cli/registry.ts`. It imports each of the six domain descriptors through an explicit import statement and exports a `CLI_DOMAINS: readonly Domain[]` enumeration. `src/cli.ts` registers domains by iterating `CLI_DOMAINS` instead of naming each one, so adding a domain edits only the registry's import list — never the composition root. The dead `src/domains/registry.ts` (the imperative `registerDomain` / `getAllDomains` Map the static enumeration replaces — never imported anywhere) was removed.

### Remaining

- Move process-boundary writes out of the command handlers into the descriptors — for example, `src/commands/session/handoff.ts:123` returns its warning text for the descriptor to emit instead of calling `process.stderr.write`.

Remove this note once every descriptor lives under `src/interfaces/cli/`, the static registry drives the composition root, and no command handler carries a process-boundary write.
