# Session Frontmatter

Every session file `spx session` writes carries the metadata an agent needs to choose and resume work. `spx session handoff` requires a non-empty goal and next step, records the git base from the current checkout, and refuses handoffs whose base cannot be reached from the shared queue with a prerequisite checklist. A caller may additionally supply an explicit work-branch ref in the stdin JSON header; `spx session handoff` records that ref as `git_ref` in place of the base it would otherwise derive, but only after confirming the branch exists on `origin`, and refuses the handoff when it does not — the git-context gate that validates the running worktree is unchanged and still governs whether the handoff is permitted at all. `spx session archive` moves any session to `archive/` without field gating, and every read command renders a session whose metadata omits structured fields as empty rather than rejecting it.

## Rationale

`goal` and `next_step` form the forward-looking handoff — the handing-off agent declares why the session exists and what to do first, and the resuming agent acts on them — so the shape carries no recorded-result field: the `archive/` directory holds sessions drained from the active queue and is never consulted for outcomes, and archive is an unconditional rename so no session is ever stranded in the active queue. Optional structured fields would rot without enforcement (sessions land with empty goals and the picker falls back to parsing the body), so `goal` and `next_step` are required at write; `created_at` and `agent_session_id` are prefilled because the agent does not know them while everything else it does. The session queue is shared across worktrees per `spx/15-worktree-management.pdr.md`, so a resuming agent in a different working copy reads the queue and must reach the session's base without the working copy that produced it — which is why no working copy is recorded (a recorded `worktree` path would invite navigation to a checkout that may not exist) and why the handoff base is constrained instead. `git_ref` defaults to a git-derived base rather than free caller text because an unverifiable caller value would let a session point at state no other checkout can reach; the caller may nonetheless supply an explicit work-branch ref, which `spx session handoff` admits only after confirming the branch exists on `origin` — an origin branch is reachable from the shared queue by exactly the standard the base gate enforces, so the verification, not blind trust, is what makes the caller value admissible. The explicit ref answers the bare-pool layout, where feature work lives on a named branch in a pool worktree the gate refuses as a base: the worktree detaches to `origin/<default>` and the caller names the pushed feature branch directly, so `git_ref` anchors at the work rather than at the unrelated base SHA. In the main checkout the derived base is the checked-out branch (or the HEAD SHA when detached), and a non-main checkout — whose branch and uncommitted state are local and cannot be reconstructed from the shared file — is permitted only from a clean checkout detached at `origin/<default>`, recording that commit's SHA rather than the default branch name because no work sits on the default branch and the SHA names the exact reachable commit. The git-context gate validates the running worktree — leaving a pool worktree clean and detached so another session can reuse it — independently of which ref is recorded: a verified work-branch ref changes what `git_ref` names, never whether the handoff is permitted. The default branch is resolved from git so the rule holds for products whose default is not `main`. The refusal is a checklist rather than a static message because an agent that hits it must learn from the diagnostic alone which prerequisite failed and how to satisfy it — the prerequisites differ between the main checkout and non-main checkouts, so a single message would force the agent to re-derive git state by hand; success stays silent because an accepted base has nothing to report, and the clean-tree remedy keeps in-progress work visible to resuming and concurrent agents while satisfying the clean-tree check. Handoff input and session-file storage stay separate so callers can supply structured fields without ambiguity while session files remain readable in markdown tools. The `specs` and `files` arrays are file paths, so a non-empty entry resolving to an existing directory is malformed input: a directory carries no injectable file content. Handoff rejects such an entry at write time, where the author can correct it, while a path that does not yet exist on disk is accepted because sessions outlive the files they reference, and an empty entry — which names no path — is recorded verbatim; pickup stays robust to whatever each entry later resolves.

Rejected: keeping the priority+tags+created_at shape and adding `goal`/`next_step` as optional fields (optional structured fields rot; required-at-write is the only durable contract); retaining a `result` field (no reader consults it and requiring it before archive strands abandoned sessions); recording the working copy as a `worktree` path (invites navigation to a checkout that may not exist in the resuming agent's environment); recording the default branch name in a detached base's `git_ref` (names a branch the session never sits on — the SHA carries the same reachability and names the exact commit); allowing a non-main-checkout handoff on any branch or a dirty tree (a worktree-local branch or dirty tree cannot be reached from the shared queue); recording a caller-supplied work-branch ref without confirming it exists on `origin` (an unverified ref reintroduces the dangling-pointer failure the git-derived base prevents — a session pointing at a branch no other checkout can fetch); letting the explicit ref bypass the worktree gate (the ref names where the work is, not whether the running worktree is safe to leave — both must hold); refusing with a single static message carrying placeholder text (forces the agent to re-derive git state and the literal `origin/<default>` names no concrete ref); and offering a clean-tree remedy that hides local work outside the shared session queue.

## Product properties

1. `spx session list` shows each session's `goal` and `next_step`, so an agent picks the work to resume without opening the file.
2. `spx session handoff` writes a session only from a base a resuming agent can reach from the shared queue — the main checkout, or a clean checkout detached at the `origin/<default>` tip — and on a non-main-checkout refusal prints a prerequisite checklist carrying resolved git values and a work-visible remedy for each unmet requirement; a caller may anchor `git_ref` at an explicit work-branch ref, which the command records only after confirming the branch exists on `origin` and refuses otherwise.
3. A session whose metadata omits structured fields stays readable by every command, and every caller-supplied string field round-trips identically for any unicode codepoint.

## Verification

The metadata field set the rules below verify:

| Field              | Type     | Lifecycle                                                                                                                                                                                                                                                                        | Default                 |
| ------------------ | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------- |
| `created_at`       | string   | Prefilled by `spx session handoff` from the system clock                                                                                                                                                                                                                         | -                       |
| `agent_session_id` | string   | Prefilled by `spx session handoff` from `$CLAUDE_SESSION_ID`, falling back to `$CODEX_THREAD_ID`                                                                                                                                                                                 | absent when neither set |
| `priority`         | string   | Set by caller at handoff; defaults to `medium` when omitted; validated as one of the `SESSION_PRIORITY` values (`high`/`medium`/`low`)                                                                                                                                           | `medium`                |
| `git_ref`          | string   | Git-derived base by default (checked-out branch name, HEAD SHA when detached, or the `origin/<default>` tip SHA for a permitted non-main checkout); a caller-supplied work-branch ref is recorded in its place once `spx session handoff` confirms the branch exists on `origin` | -                       |
| `goal`             | string   | Provided by caller at handoff; validated non-empty                                                                                                                                                                                                                               | -                       |
| `next_step`        | string   | Provided by caller at handoff; validated non-empty                                                                                                                                                                                                                               | -                       |
| `specs`            | string[] | Optional file paths for auto-injection on pickup; a non-empty entry resolving to an existing directory is rejected at handoff                                                                                                                                                    | `[]`                    |
| `files`            | string[] | Optional file paths for auto-injection on pickup; a non-empty entry resolving to an existing directory is rejected at handoff                                                                                                                                                    | `[]`                    |

### Testing

- ALWAYS: `spx session handoff` rejects piped content lacking a non-empty `goal` and a non-empty `next_step` — they are the handoff payload ([compliance])
- ALWAYS: `spx session handoff` rejects a non-empty `specs` or `files` entry that resolves to an existing directory, writing no file and exiting non-zero with `SessionInjectionDirectoryError` naming the entry; an entry that does not exist on disk is accepted, since sessions outlive the files they reference, and an empty entry names no path and is recorded verbatim ([compliance])
- NEVER: `spx session handoff` records a non-empty `specs` or `files` entry that resolves to an existing directory — the arrays are file paths and a directory carries no injectable content ([compliance])
- ALWAYS: `spx session handoff` in the main checkout records the checked-out branch name, or the HEAD commit SHA when detached, as the session base, unless the caller supplies a work-branch ref the command has verified exists on `origin`; caller metadata otherwise cannot override that base ([compliance])
- ALWAYS: `spx session handoff` in a non-main checkout refuses unless the working tree is clean and HEAD is detached at the tip of `origin/<default>`, exiting non-zero with the prerequisite checklist; on success it records that commit SHA as the session base, unless the caller supplies a work-branch ref the command has verified exists on `origin` ([compliance])
- ALWAYS: `spx session handoff` enforces the non-main-checkout base against the repository's actual default-branch tip, whatever the default branch is named ([compliance])
- ALWAYS: when the caller supplies a work-branch ref in the stdin JSON header, `spx session handoff` confirms the branch exists on `origin` as an exact remote-tracking ref — not a resolvable revision expression — and records that ref as `git_ref`, overriding the base it would otherwise derive ([compliance])
- NEVER: `spx session handoff` accepts a caller `git_ref` that resolves only as a revision expression (e.g. `main~1`, `main^{commit}`, a bare SHA) rather than an exact `origin` branch — such a ref is refused with `SessionWorkBranchNotOnOriginError` ([compliance])
- NEVER: `spx session handoff` accepts a caller `git_ref` of `HEAD` — `refs/remotes/origin/HEAD` is the symbolic default-branch pointer, not a work branch, so it is refused with `SessionWorkBranchNotOnOriginError` ([compliance])
- ALWAYS: an absent or empty-string `git_ref` is treated as no ref supplied — `spx session handoff` records the gate-derived base and runs no origin probe ([compliance])
- NEVER: `spx session handoff` records a caller-supplied work-branch ref that does not exist on `origin` — it refuses the handoff with `SessionWorkBranchNotOnOriginError` ([compliance])
- NEVER: a caller-supplied work-branch ref bypasses the git-context gate — a linked worktree that is not clean and detached at the `origin/<default>` tip is refused regardless of the ref ([compliance])
- ALWAYS: `spx session handoff` prints no base diagnostic on a permitted handoff — success emits only the session's `<HANDOFF_ID>` and `<SESSION_FILE>` tags ([compliance])
- ALWAYS: on a non-main-checkout refusal, the diagnostic renders one checklist line per base prerequisite marked met or unmet, shown only when at least one prerequisite is unmet ([compliance])
- ALWAYS: the handoff refusal checklist carries resolved git values — the resolved default branch, the `origin/<default>` tip commit SHA, the observed HEAD commit SHA, the current worktree path, and the main-checkout path — stating a value as unresolved when git cannot resolve it, rather than fabricating a placeholder, in a layout of the renderer's choice ([compliance])
- ALWAYS: when `origin/HEAD` is unset so `origin/<default>` resolves to no commit, the checklist marks the detached-at-tip prerequisite unmet and states the default branch and its `origin/<default>` tip as unresolved rather than a fabricated branch name, SHA, or the literal-text placeholder ([compliance])
- ALWAYS: each unmet checklist line states its failure reason and a remedy — the unclean-tree remedy is to commit the changes or run handoff from the main checkout, the off-tip remedy is to detach HEAD to the `origin/<default>` tip or run handoff from the main checkout, and the unresolved-base remedy (when `origin/HEAD` is unset) is to run handoff from the main checkout ([compliance])
- ALWAYS: `spx session handoff` prefills `created_at` as an ISO-8601 timestamp with timezone offset per `spx/36-session.enabler/21-timestamp-format.adr.md` ([compliance])
- ALWAYS: `spx session handoff` prefills `agent_session_id` from `$CLAUDE_SESSION_ID` when set, falling back to `$CODEX_THREAD_ID`; when neither is set the field is absent ([compliance])
- ALWAYS: `spx session handoff` defaults `priority` to `medium` when the caller omits it and validates it as one of the `SESSION_PRIORITY` values (`high`, `medium`, `low`) ([compliance])
- ALWAYS: `spx session archive` moves a session from `todo/` or `doing/` to `archive/` by rename for any resolvable session id, validating no frontmatter field ([compliance])
- ALWAYS: `spx session list`, `show`, `pickup`, and `release` render missing structured fields as empty strings without rejecting the session ([compliance])
- ALWAYS: `spx session handoff` preserves every string-typed metadata field exactly, including punctuation, quotes, whitespace, and newlines ([property])
- ALWAYS: `spx session handoff` accepts caller-supplied structured fields before the body, consumes at most one immediately following line separator, and treats the remaining bytes as the body verbatim ([compliance])
- NEVER: `spx session handoff` substitutes a default body for empty piped content — empty handoffs are rejected with `SessionInvalidContentError` ([compliance])
- NEVER: `spx session handoff` writes a metadata key outside the field set declared in the table above ([compliance])
- NEVER: `spx session handoff` lets caller metadata override the session base, except a work-branch ref the command has verified exists on `origin` ([compliance])
- NEVER: `spx session handoff` invents a value for `agent_session_id` when both environment variables are absent — the field is omitted, not populated with a placeholder ([compliance])
- NEVER: `spx session handoff` parses caller-supplied stdin as session-file frontmatter — input opening with the frontmatter delimiter is rejected with `SessionLegacyFrontmatterInputError` ([compliance])
- NEVER: `spx session archive` rejects a session for a missing or empty metadata field — archive is an unconditional move ([compliance])
- NEVER: any command embeds session lineage (`previous_session_id`, `previous_result`) in the metadata — each session carries its own `goal` and `next_step` ([compliance])
- NEVER: the handoff refusal remedy tells the user to hide uncommitted work outside the shared session queue; remedies are to commit the work, detach to the default-branch tip, or run handoff from the main checkout ([compliance])
- NEVER: the handoff refusal diagnostic renders the literal-text placeholder `origin/<default>` in place of a base ref that resolves to a commit — a resolvable ref is shown by its value, an unresolvable one is stated as unresolved ([compliance])
- NEVER: the handoff refusal checklist omits any base prerequisite — every prerequisite appears on its own line with its met/unmet marker ([compliance])
