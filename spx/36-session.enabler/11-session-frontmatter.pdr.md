# Session Frontmatter

Every session file `spx session` writes carries a fixed YAML frontmatter shape. `spx session handoff` validates non-empty `goal` and `next_step`, sources `git_ref` from git context rather than caller input, and writes a session only from a base a resuming agent can reach from the shared queue — refusing a linked-worktree handoff that is not a clean checkout detached at `origin/<default>` with a prerequisite checklist. `spx session archive` moves any session to `archive/` by rename without gating on any field, and every read command renders a session whose frontmatter omits structured fields as empty rather than rejecting it.

## Rationale

`goal` and `next_step` form the forward-looking handoff — the handing-off agent declares why the session exists and what to do first, and the resuming agent acts on them — so the shape carries no recorded-result field: the `archive/` directory holds sessions drained from the active queue and is never consulted for outcomes, and archive is an unconditional rename so no session is ever stranded in the active queue. Optional structured fields would rot without enforcement (sessions land with empty goals and the picker falls back to parsing the body), so `goal` and `next_step` are required at write; `created_at` and `agent_session_id` are prefilled because the agent does not know them while everything else it does. The session queue is shared across worktrees per `spx/15-worktree-resolution.pdr.md`, so a resuming agent in a different working copy reads the queue and must reach the session's base without the working copy that produced it — which is why no working copy is recorded (a recorded `worktree` path would invite navigation to a checkout that may not exist) and why the handoff base is constrained instead. `git_ref` is sourced from git rather than the caller because the caller's value is unverifiable; in the root worktree it records the checked-out branch (or the HEAD SHA when detached), and a linked worktree — whose branch and uncommitted state are local and cannot be reconstructed from the shared file — is permitted only from a clean checkout detached at `origin/<default>`, recording that commit's SHA rather than the default branch name because no work sits on the default branch and the SHA names the exact reachable commit. The default branch is resolved from git so the rule holds for products whose default is not `main`. The refusal is a checklist rather than a static message because an agent that hits it must learn from the diagnostic alone which prerequisite failed and how to satisfy it — the prerequisites differ between root and linked worktrees, so a single message would force the agent to re-derive git state by hand; success stays silent because an accepted base has nothing to report, and the clean-tree remedy excludes stashing because a stash hides in-progress work from the resuming and concurrent agents while satisfying the literal check. The input wire format is a JSON object at the start of stdin because every caller-supplied scalar is unambiguously quoted, with no plain-scalar or comment ambiguity; the on-disk format stays YAML so markdown-aware tools fold the frontmatter, and the two formats need not be the same.

Rejected: keeping the priority+tags+created_at shape and adding `goal`/`next_step` as optional fields (optional structured fields rot; required-at-write is the only durable contract); retaining a `result` field (no reader consults it and requiring it before archive strands abandoned sessions); recording the working copy as a `worktree` path (invites navigation to a checkout that may not exist in the resuming agent's environment); recording the default branch name in a detached base's `git_ref` (names a branch the session never sits on — the SHA carries the same reachability and names the exact commit); allowing a linked-worktree handoff on any branch or a dirty tree (a worktree-local branch or dirty tree cannot be reached from the shared queue); refusing with a single static message carrying placeholder text (forces the agent to re-derive git state and the literal `origin/<default>` names no concrete ref); and offering stashing as the clean-tree remedy (the stash stays in the producing worktree, invisible to the resuming and concurrent agents).

## Product properties

1. `spx session list` shows each session's `goal` and `next_step`, so an agent picks the work to resume without opening the file.
2. `spx session handoff` writes a session only from a base a resuming agent can reach from the shared queue — the root worktree, or a clean checkout detached at the `origin/<default>` tip — and on a linked-worktree refusal prints a prerequisite checklist carrying resolved git values and a non-stashing remedy for each unmet requirement.
3. A session whose frontmatter omits structured fields stays readable by every command, and every caller-supplied string field round-trips identically for any unicode codepoint.

## Verification

The frontmatter shape the rules below verify:

| Field              | Type     | Lifecycle                                                                                                                              | Default                 |
| ------------------ | -------- | -------------------------------------------------------------------------------------------------------------------------------------- | ----------------------- |
| `created_at`       | string   | Prefilled by `spx session handoff` from the system clock                                                                               | -                       |
| `agent_session_id` | string   | Prefilled by `spx session handoff` from `$CLAUDE_SESSION_ID`, falling back to `$CODEX_THREAD_ID`                                       | absent when neither set |
| `priority`         | string   | Set by caller at handoff; defaults to `medium` when omitted; validated as one of the `SESSION_PRIORITY` values (`high`/`medium`/`low`) | `medium`                |
| `git_ref`          | string   | Prefill-only from git context — the checked-out branch name, or the HEAD commit SHA when detached; caller value ignored                | -                       |
| `goal`             | string   | Provided by caller at handoff; validated non-empty                                                                                     | -                       |
| `next_step`        | string   | Provided by caller at handoff; validated non-empty                                                                                     | -                       |
| `specs`            | string[] | Optional file paths for auto-injection on pickup                                                                                       | `[]`                    |
| `files`            | string[] | Optional file paths for auto-injection on pickup                                                                                       | `[]`                    |

### Audit

- ALWAYS: `spx session handoff` rejects piped content lacking a non-empty `goal` and a non-empty `next_step` — they are the handoff payload ([audit])
- ALWAYS: `spx session handoff` in the root worktree prefills `git_ref` from `git rev-parse --abbrev-ref HEAD`, or the HEAD commit SHA when detached; the value is prefill-only and a caller-supplied `git_ref` is ignored ([audit])
- ALWAYS: `spx session handoff` in a linked worktree refuses unless the working tree is clean and HEAD is detached at the tip of `origin/<default>`, exiting non-zero with `SessionHandoffBaseError` rendered as the prerequisite checklist; on success it records `git_ref` as that commit SHA ([audit])
- ALWAYS: `spx session handoff` resolves the default branch from git rather than a hardcoded name when enforcing the linked-worktree base ([audit])
- ALWAYS: `spx session handoff` prints no base diagnostic on a permitted handoff — success emits only the session's `<HANDOFF_ID>` and `<SESSION_FILE>` tags ([audit])
- ALWAYS: on a linked-worktree refusal, `SessionHandoffBaseError` renders one checklist line per base prerequisite marked met or unmet, shown only when at least one prerequisite is unmet ([audit])
- ALWAYS: the handoff refusal checklist carries resolved git values — the resolved default branch, the `origin/<default>` tip commit SHA, the observed HEAD commit SHA, the current worktree path, and the root-worktree path — rather than unresolved placeholders, in a layout of the renderer's choice ([audit])
- ALWAYS: when `origin/HEAD` is unset so `origin/<default>` resolves to no commit, the checklist marks the detached-at-tip prerequisite unmet and states the default branch and its `origin/<default>` tip as unresolved rather than a fabricated branch name, SHA, or the literal-text placeholder ([audit])
- ALWAYS: each unmet checklist line states its failure reason and a remedy — the unclean-tree remedy is to commit the changes or run handoff from the root worktree, the off-tip remedy is to detach HEAD to the `origin/<default>` tip or run handoff from the root worktree, and the unresolved-base remedy (when `origin/HEAD` is unset) is to run handoff from the root worktree ([audit])
- ALWAYS: `spx session handoff` prefills `created_at` as an ISO-8601 timestamp with timezone offset per `spx/36-session.enabler/21-timestamp-format.adr.md` ([audit])
- ALWAYS: `spx session handoff` prefills `agent_session_id` from `$CLAUDE_SESSION_ID` when set, falling back to `$CODEX_THREAD_ID`; when neither is set the field is absent ([audit])
- ALWAYS: `spx session handoff` defaults `priority` to `medium` when the caller omits it and validates it as one of the `SESSION_PRIORITY` values (`high`, `medium`, `low`) ([audit])
- ALWAYS: `spx session archive` moves a session from `todo/` or `doing/` to `archive/` by rename for any resolvable session id, validating no frontmatter field ([audit])
- ALWAYS: `spx session list`, `show`, `pickup`, and `release` render missing structured fields as empty strings without rejecting the session ([audit])
- ALWAYS: `spx session handoff` writes every string-typed frontmatter field through the `yaml` package's `stringify` so `git_ref`, `agent_session_id`, `goal`, `next_step`, and `created_at` values containing YAML-special characters (`:`, `{`, `}`, `#`, `|`, `\`, quotes, spaces, newlines) round-trip cleanly through `parseSessionMetadata` ([audit])
- ALWAYS: `spx session handoff` accepts caller-supplied structured fields as a JSON object at the start of stdin, consumes at most one immediately following `LF` or `CRLF` separator, and treats the remaining bytes as the body verbatim ([audit])
- NEVER: `spx session handoff` substitutes a default body for empty piped content — empty handoffs are rejected with `SessionInvalidContentError` ([audit])
- NEVER: `spx session handoff` writes a frontmatter key outside the shape declared in the table above ([audit])
- NEVER: `spx session handoff` accepts a caller-supplied `git_ref` — it is sourced from git context regardless of input ([audit])
- NEVER: `spx session handoff` invents a value for `agent_session_id` when both environment variables are absent — the field is omitted, not populated with a placeholder ([audit])
- NEVER: `spx session handoff` parses caller-supplied stdin as YAML — input opening with the YAML-frontmatter delimiter is rejected with `SessionLegacyFrontmatterInputError` ([audit])
- NEVER: `spx session archive` rejects a session for a missing or empty frontmatter field — archive is an unconditional move ([audit])
- NEVER: any command embeds session lineage (`previous_session_id`, `previous_result`) in the frontmatter — each session carries its own `goal` and `next_step` ([audit])
- NEVER: the handoff refusal remedy suggests `git stash` — stashed work is hidden from the resuming and concurrent agents ([audit])
- NEVER: the handoff refusal diagnostic renders the literal-text placeholder `origin/<default>` in place of a base ref that resolves to a commit — a resolvable ref is shown by its value, an unresolvable one is stated as unresolved ([audit])
- NEVER: the handoff refusal checklist omits any base prerequisite — every prerequisite appears on its own line with its met/unmet marker ([audit])
